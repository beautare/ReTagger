# CLAUDE.md

本文件为在本仓库中协作的 Claude 提供行动准则。

## 基本规则
- 总是用中文进行回复，保持专业且明确的语气。
- 尽量避免写兼容代码，聚焦当前目标平台的最佳实现。
- 对外暴露的信息以用户价值为先，避免泄露无关内部细节。
- 不要防御性编程。

## 项目愿景
ReTagger 是一款面向音乐工作者的 macOS SwiftUI 应用，用于批量整理 MP3、FLAC、AAC / ALAC（M4A）、WAV、AIFF、Ogg、Opus 等主流音频元数据、统一音轨命名并重构目录结构。产品核心在于“智能提炼 + 安全落盘”：以 AI 辅助生成规范化标签，同时保证本地文件操作的可控与可追溯。

## 技术栈与架构基线
- **平台定位**：原生 macOS 应用，依托 SwiftUI 与现代 Swift 并发模型，默认主线程驱动 UI，后台服务承担文件与网络操作；最低支持 macOS 12.4。
- **前后端协作**：本项目为前端应用，对应的后端服务为独立的 Intelli-Sight 项目（闭源，未随本仓库发布），前端通过调用后端 API 完成 AI 推理、元数据处理等核心功能；后端地址通过 `.env` / `.env.local` 配置。
- **目录划分**：`Features` 面向场景的界面组合，`Views` 提供通用视图，与 `Core` 中的服务、模型、协调器形成清晰分层；设计语言统一由 `Core/DesignSystem` 输出。
- **服务协作**：文件系统、元数据缓存、网络 AI 适配等服务通过协议抽象连接，强调可替换性与最小依赖；批处理流程遵循"采集 → 洗数 → AI 推理 → 校验落盘"的串行节点。
- **数据边界**：所有外部资源（本地文件、AI 接口）进入系统前需标准化结构并标记来源，保持状态可回滚、操作可追踪。
- 左侧侧边栏顶部的"添加目录"图标按钮需触发目录选择，右侧“历史”图标菜单滚动展示最近 50 个目录，仅以路径呈现，数据来源于 `AppCoordinator.recentDirectories`。
- `Features/DirectorySelection` 中"选择目录"按钮下默认展示最近 5 个目录（仅路径可点击），依赖 `AppSettings.recentDirectories` 持久化。
- **沙盒权限管理**：所有目录扫描的统一入口为 `DirectorySelectionView.performScan`，在该方法内通过 `AppCoordinator.activateSecurityScope` 激活并持久化权限。权限会一直保持直到下一次目录选择或调用 `reset` 方法释放旧权限，确保播放、元数据操作等后续流程不丢失沙盒授权。

## 设计原则
- **目标驱动**：先定义用户任务，再决定界面和服务的组合方式；避免为未知场景预埋兼容层。
- **分层清晰**：视图只关心展示与交互，状态协调交给 Feature 级容器，业务决策沉淀在 Core 服务。
- **约束优先**：在命名、批处理规则、文件改写策略上保持一致，所有约束写入常量或策略对象而非散落条件判断。
- **复用与组合**：复用现有组件与设计系统，若需新能力，优先通过组合扩展而非复制粘贴。
- **可观察性**：关键操作应产生日志与可视化状态，用于排查 AI 结果或文件写入差异。

## 编码原则
- **D.R.Y**：优先抽取可复用的批处理逻辑、日志策略和视图组件，避免在不同 Feature 间重复实现。
- **S.O.L.I.D**：保持单一职责、依赖倒置和接口隔离，使用协议注入替代硬编码依赖，便于后续替换服务实现。
- **不可变优先**：默认使用不可变数据结构，明确标注需要共享或持久化的可变状态。
- **显式错误处理**：使用可追踪的错误模型和恢复策略，禁止吞掉异常或依赖隐式回退。

## 协作与评审
- **迭代节奏**：功能切分以完整用户路径或核心服务变更为单位，提交前确保上下游影响已评估。
- **变更叙述**：提交信息保持简洁、中文、面向结果；描述场景、调整的模块以及对用户的直接收益。
- **评审重点**：关注架构契合度、并发安全、数据落盘策略、防御性编程；对潜在风险提出阻塞性意见。

- **文档同步**：若约束或技术栈发生变更，优先更新 `AGENTS.md`、`CLAUDE.md` 这两份文档，确保项目定位、架构原则与流程要求完全一致。

## 质量保障
- **测试视角**：以真实音乐库为样本覆盖目录深度、字符集混杂、AI 失败回退等场景；重点验证批处理的失败隔离和结果回滚。
- **体验守护**：界面响应需保持流畅，长耗时操作提供进度反馈与取消入口；所有用户提示使用一致的语气和设计系统样式。
- **发布准备**：版本冻结前复盘架构变化、AI 能力表现、文档准确性，确保知情人能快速掌握当前系统边界。
